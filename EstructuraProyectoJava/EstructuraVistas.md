# Estructura Estándar para Vistas (JDialog)

## Patrones Generales

### 1. Nomenclatura

- **Clase**: `[Entidad]Vista` (ej: `UsuariosVista`, `ClientesVista`, `ProductosVista`)
- **Archivos**: Siempre se crean `.java` y `.form` para NetBeans
- **Extensión**: Siempre heredan de `javax.swing.JDialog`

### 2. Estructura Visual Básica

Toda vista contendrá los siguientes componentes obligatorios:

#### Encabezado

- **Componente**: `jLabel` con texto grande
- **Formato**: Arial Black, 24px, color (#504F62)
- **Contenido**: Nombre de la entidad en MAYÚSCULAS (ej: "USUARIOS", "CLIENTES", "PRODUCTOS")

#### Tabla Principal

- **Nombre**: `tabla` (siempre público)
- **Tipo**: `JTable`
- **Contenido**: Columnas relacionadas con la entidad
- **Ubicación**: Parte central de la vista

#### Buscador

- **Nombre**: `inputBusqueda` o similar
- **Tipo**: `JTextField`
- **Botón**: `botonBuscar` (texto: "buscar")

#### Botones de Acción

- **Agregar**: `botonAgregar` (texto: "Agregar")
- **Informe**: `botonInforme` (texto: "Ver Informe") - solo si corresponde

### 4. Convenciones de Nomenclatura

#### Componentes Públicos

- **Tablas**: `tabla` (única tabla principal)
- **Campos de texto**: `input[Campo]` (ej: `inputNombreUsuario`, `inputRTNCliente`)
- **Combobox**: `comboBox[Entidad]` (ej: `comboBoxProductos`, `comboBoxRoles`)
- **Botones**: `boton[Acción]` (ej: `botonBuscar`, `botonAgregar`, `botonInforme`)
- **Labels**: `jLabel[Numero]` (generados por NetBeans)

#### Campos Automáticos (NO incluir en formularios)

- IDs autogenerados: `id_usuario`, `id_cliente`, `id_producto`
- Timestamps: `fecha_creacion`, `fecha_registro`
- Estados por defecto: `estado_usuario`, `estado_cliente`

### 6. Principios MVC

- **Solo presentación**: La vista no contiene lógica de negocio
- **Componentes públicos**: Todos los componentes son accesibles desde el controlador
- **Sin validaciones**: Las validaciones se manejan en el controlador
- **Sin conexión a BD**: La vista nunca accede directamente a la base de datos

### 7. Proceso de Creación

1. **Revisar init.sql**: Identificar campos de la tabla
2. **Revisar Type classes**: Conocer estructura de datos
3. **Crear archivos**: `[Entidad]Vista.java` y `[Entidad]Vista.form`
4. **Diseñar visualmente**: Usar NetBeans GUI Designer
5. **Aplicar nomenclatura**: Seguir convenciones establecidas
6. **Hacer públicos**: Todos los componentes deben ser públicos
