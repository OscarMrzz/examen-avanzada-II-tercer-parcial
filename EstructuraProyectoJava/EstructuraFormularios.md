# Estructura Estándar para Formularios (JDialog)

## Patrones Generales

### 1. Nomenclatura

- **Clases**:
  - `FormularioAgregar[Entidad]` (ej: `FormularioAgregarUsuario`)
  - `FormularioEditar[Entidad]` (ej: `FormularioEditarUsuario`)
- **Archivos**: Siempre se crean `.java` y `.form` para NetBeans
- **Extensión**: Siempre heredan de `javax.swing.JDialog`

### 2. Estrategia de Creación

- **Paso 1**: Crear `FormularioAgregar[Entidad]` completo
- **Paso 2**: Copiar y pegar para crear `FormularioEditar[Entidad]`
- **Paso 3**: Refactorizar nombre de clase y título
- **Resultado**: Dos formularios idénticos en estructura, diferente título

### 3. Estructura Visual Básica

#### Título del Formulario

- **Componente**: `jLabel` principal
- **Formato**: Arial Black, 14px
- **Contenido**: "Agregar [Entidad]" o "Editar [Entidad]"

#### Campos de Entrada

- **Nomenclatura**: `input[Campo]` (ej: `inputNombreUsuario`, `inputRTNCliente`)
- **Tipos**:
  - `JTextField` para texto simple
  - `JTextArea` para texto largo
  - `JComboBox` para selecciones
- **Labels**: Descriptivos para cada campo

#### Botones de Acción

- **Guardar**: `botonGuardar` (texto: "Guardar")
- **Cancelar**: `botonCancelar` (texto: "Cancelar")
- **Imágenes**: `botonAgregarImagen` si aplica

### 5. Convenciones de Nomenclatura

#### Campos de Entrada Públicos

- **Texto**: `input[Campo]` (ej: `inputNombreUsuario`, `inputPassword`)
- **Combobox**: `comboBox[Entidad]` (ej: `comboBoxRol`, `comboBoxTipoCliente`)
- **Áreas de texto**: `inputArea[Descripcion]` (ej: `inputAreaDireccion`)
- **Botones**: `boton[Acción]` (ej: `botonGuardar`, `botonCancelar`)

#### Campos Automáticos (NO incluir)

- IDs autogenerados: `id_usuario`, `id_cliente`, `id_producto`
- Timestamps: `fecha_creacion`, `fecha_registro`
- Estados por defecto: `estado_usuario`, `estado_cliente`
- Campos calculados: `total_venta`, `subtotal_venta`

### 6. Proceso de Creación por Entidad

#### Paso 1: Análisis de la Base de Datos

```sql
-- Revisar tabla en init.sql
DESCRIBE usuarios;
-- Campos: id_usuario, nombre_usuario, user_usuario, pass_usuario,
-- privilegio_usuario, foto_usuario, fecha_creacion, estado_usuario
```

#### Paso 2: Identificar Campos Manuales

- **Incluir**: Campos que el usuario debe ingresar
- **Excluir**: Campos automáticos (IDs, timestamps, estados)

#### Paso 3: Revisar Type Classes

```java
// Revisar Type/[Entidad]Type.java

    private String privilegio_usuario; // Sí incluir -> comboBoxRol

```

### 8. Principios MVC

#### Responsabilidades del Formulario

- **Solo presentación**: Diseño visual y recolección de datos
- **Sin validaciones**: Las validaciones se manejan en el controlador
- **Sin lógica de negocio**: No calcular totales ni aplicar reglas
- **Sin conexión a BD**: Nunca acceder directamente a la base de datos

#### Acceso desde Controlador

- **Componentes públicos**: Todos los campos son accesibles
- **Getter/Setter implícitos**: Acceso directo a los valores
- **Eventos**: El controlador registra listeners en los botones

### 9. Proceso de Creación Detallado

#### Paso 1: Investigación

```bash
# 1. Revisar init.sql para la tabla
grep -A 20 "CREATE TABLE usuarios" src/Modelo/init.sql

# 2. Revisar Type class
cat src/Type/UsuarioType.java
```

#### Paso 2: Diseño del Formulario

1. **Crear archivos**: `FormularioAgregar[Entidad].java` y `.form`
2. **Diseñar visualmente**: Arrastrar componentes al formulario
3. **Aplicar nomenclatura**: Renombrar componentes siguiendo convenciones
4. **Hacer públicos**: Asegurar que todos los componentes sean públicos

#### Paso 3: Creación del Formulario de Edición

1. **Copiar archivos**: Duplicar archivos .java y .form
2. **Refactorizar clase**: Cambiar nombre a `FormularioEditar[Entidad]`
3. **Actualizar título**: Cambiar "Agregar [Entidad]" a "Editar [Entidad]"
4. **Verificar estructura**: Mantener misma disposición de componentes

### 12. Campos Especiales

#### Imágenes

- **Componente**: `botonAgregarImagen`
- **Panel**: `jPanel1` con `labelImagem` para vista previa
- **Funcionalidad**: Solo UI, la lógica va en controlador

#### Fechas

- **Componente**: `inputFecha[Campo]` con `JFormattedTextField`
- **Formato**: dd/MM/yyyy o similar según requerimiento

#### Números y Decimales

- **Componente**: `input[Campo]` con validación de formato
- **Precisión**: Según definición en init.sql (DECIMAL, INT)

#### Textos Largos

- **Componente**: `inputArea[Descripcion]` con `JTextArea`
- **Scroll**: Incluir `JScrollPane` para textos largos
